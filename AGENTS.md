# AGENTS.md — Presentation Template

## Project Overview

This project contains a **25-minute talk** titled **"OpenClaw Optimization: Make Your AI Agent Cheaper, Faster, and Smarter"**.

The talk targets a mixed technical and non-technical audience — developers and product managers who use or are considering OpenClaw as an AI agent platform. The goal is to show how tuning `openclaw.json` can cut AI agent running costs by 60–80% with no code changes and no quality loss.

**Key message:** Eight configurations across four groups — Context Budget, Memory, Model Routing, and Heartbeat Scheduling — are the highest-impact levers. Each is explained simply and is immediately applicable.

## How to Build

```
npm run build    # renders both language versions into output/
npm run preview  # opens output/index.html (English) in the browser
```

Requires node 20 (pinned via `mise.toml`). Run `mise install` if switching environments.

### Output files

| File | Language |
|---|---|
| `output/index.html` | English (default) |
| `output/index.vi.html` | Vietnamese |

Each file includes a floating language-switcher button linking to the other version.

## Git Commit Convention

Format: `<scope>: <short description>`

## Presentation Content Rules

- **Slide content is short and punchy** — audience-facing, not speaker notes
- **Speaker notes are in HTML comments** — verbose, narrative, story-driven
- **No live demo** — config examples and tables only
- **Never edit `output/index.html` or `output/index.vi.html` directly** — they are generated by `build.cjs`
- **Non-tech explanations are required** — every config must have a plain-English analogy before the code snippet

## Workflow

Always follow these steps when changing the presentation:

1. Plan the change
2. Make the change in `slides/presentation.en.md` AND `slides/presentation.vi.md`, or `build.cjs`
3. Run `npm run build` to render into HTML
4. Verify with visual QA — open `output/index.html` and `output/index.vi.html` in the browser and check every slide
5. (Optional) If working in a cloud environment or commenting on a PR, include 3-5 screenshots
